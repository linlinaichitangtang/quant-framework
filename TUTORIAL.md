## 2. 快速启动

### 2.1 环境要求

| 组件 | 最低要求 | 推荐 |
|------|----------|------|
| Docker | 20.10+ | 最新版 |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 10GB | 20GB+ |
| 系统 | macOS/Windows/Linux | 最新 |

### 2.2 启动命令

```bash
# 进入项目目录
cd quant-framework

# 一键启动（首次启动需要拉取镜像，约 5-10 分钟）
./start.sh

# 查看服务状态
docker-compose ps

# 查看实时日志
docker-compose logs -f
```

| 前端界面 | http://localhost | admin / admin123 |
| API 文档 | http://localhost:8000/docs | - |
| 健康检查 | http://localhost:8000/health | - |

## 3. 界面功能

### 3.1 Dashboard 仪表盘

展示整体账户状态和关键指标：

- **资产概览**：总资产、可用资金、持仓市值
- **今日收益**：当日盈亏、收益率
- **在运行策略**：当前激活的策略数量
- **信号统计**：今日/本周生成的信号数量
- **持仓分布**：各板块/股票的持仓占比

### 3.2 选股模块

路径：**选股** → **生成信号**

1. 选择策略类型（A股尾盘 / 港美股期权）
2. 设置选股参数（可选，使用默认配置）
3. 点击 **"生成信号"**
4. 等待分析完成，查看结果

**信号详情包含**：
- 股票代码、名称
- 入选理由（具体条件匹配）
- 技术指标数值
- 风控评分
- 建议买入价

### 3.3 回测模块

路径：**回测** → **运行回测**

1. 选择策略
2. 设置回测参数：
   - 初始资金
   - 手续费率
   - 滑点
   - 回测时间范围
3. 点击 **"开始回测"**
4. 查看回测报告

**回测报告指标**：

| 指标 | 说明 | 优秀值 |
|------|------|--------|
| 年化收益率 | 平均每年的收益 | > 15% |
| 最大回撤 | 历史最大亏损 | < 20% |
| 夏普比率 | 风险调整收益 | > 1.5 |
| 胜率 | 盈利交易占比 | > 50% |
| 盈亏比 | 平均盈利/亏损 | > 1.5 |
| 交易次数 | 总交易笔数 | > 100 |

### 3.4 持仓管理

路径：**持仓**

- 实时持仓列表
- 持仓盈亏计算
- 手动平仓功能
- 持仓历史查询

### 3.5 交易记录

路径：**交易记录**

- 全部交易历史
- 按时间/股票筛选
- 导出 CSV/Excel

### 3.6 系统设置

路径：**设置**

- **个人设置**：修改密码、通知偏好
- **API 配置**：数据源 Token、券商账号
- **风控设置**：止盈止损参数
- **通知设置**：企业微信/钉钉 Webhook

## 4. 策略配置

### 4.1 A 股尾盘策略

适用于 A 股超短线交易，尾盘买入次日卖出。

**选股条件**：

| 条件 | 说明 | 默认值 |
|------|------|--------|
| 均线多头 | 股价站在 20 日均线上方 | 开启 |
| 涨幅范围 | 当日涨幅 3%-8% | 3%~8% |
| 量比要求 | 当日量比 > 1.5 | 1.5 |
| 换手率 | 换手率 3%-20% | 3%~20% |
| 流通市值 | 50 亿-500 亿 | 50~500 亿 |
| 排除项 | ST、停牌、连续涨停 | 排除 |

**风控规则**：

| 规则 | 说明 |
|------|------|
| 止盈 | 次日涨幅 +2% 立即卖出 |
| 止损 | 次日跌幅 -2% 立即卖出 |
| 早盘止损 | 开盘低开 -2% 立即卖出 |
| 绝对止损 | 14:30 未触及止盈止损，无论盈亏卖出 |
| 单票上限 | 不超过总资金 10% |
| 日新仓上限 | 每日最多新开 5 只 |
| 总持仓上限 | 总持仓不超过 80% |

### 4.2 港美股期权策略

适用于事件驱动型期权交易。

**支持策略**：

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| Long Call | 买入看涨期权 | 强烈看多 |
| Long Put | 买入看跌期权 | 强烈看空 |
| Covered Call | 备兑看涨期权 | 温和看多 |
| Bull Spread | 牛市价差 | 温和看多 |
| Bear Spread | 熊市价差 | 温和看空 |
| Straddle | 跨式策略 | 波动率爆发 |
| Strangle | 宽跨式策略 | 波动率爆发 |
| Iron Condor | 铁鹰策略 | 震荡行情 |

**风控规则**：

| 规则 | 说明 |
|------|------|
| 单策略最大亏损 | 不超过总资金 3% |
| 总权利金上限 | 不超过总资金 20% |
| 权利金止损 | 亏损 50% 止损 |
| 宏观风控 | VIX > 25 暂停开仓 |

### 4.3 自定义策略

通过 YAML 配置文件自定义策略：

```yaml
# strategies/my_strategy.yaml
name: 我的自定义策略
type: a_stock

selection:
  price_above_ma: [20]
  ma_direction: up
  change_pct: [3, 8]
  volume_ratio: 1.5
  turnover_rate: [3, 20]
  market_cap: [50, 500]

risk:
  stop_profit: 0.02      # 2% 止盈
  stop_loss: 0.02        # 2% 止损
  max_position: 0.1      # 单票 10%
  max_new_positions: 5   # 日新仓 5 个
```

## 5. 回测分析

### 5.1 运行回测

```python
# via API
POST /api/backtest
{
    "strategy_id": "a_stock_short_term",
    "initial_capital": 1000000,
    "commission": 0.0003,
    "slippage": 0.001,
    "start_date": "2024-01-01",
    "end_date": "2024-12-31"
}
```

### 5.2 回测参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| initial_capital | 初始资金 | 100 万 |
| commission | 手续费率 | 0.03% |
| slippage | 滑点 | 0.1% |
| position_limit | 持仓上限 | 80% |

### 5.3 回测报告解读

**收益曲线**：
- 累计收益率 vs 基准（沪深 300）
- 回撤曲线
- 滚动夏普比率

**统计分析**：
- 总交易次数
- 盈利交易 / 亏损交易
- 平均持仓时间
- 月度收益分布

**交易明细**：
- 每笔交易的买入/卖出时间、价格
- 持仓期间的最高/最低价
- 关联的选股理由

## 6. 实盘对接

### 6.1 FMZ 发明者量化

1. **注册 FMZ 账号**：https://www.fmz.com
2. **创建策略**：在 FMZ 创建自定义策略
3. **获取 API Key**：
   - FMZ → 设置 → API Key
   - 记录 Access Key 和 Secret Key
4. **配置到系统**：设置 → API 配置 → 填写 FMZ 信息
5. **绑定机器人**：将系统与 FMZ 机器人关联

**实盘注意事项**：
- 先在模拟盘测试充足时间
- 初始仓位从小到大逐步加
- 设置合理的单日最大亏损
- 保持网络稳定

### 6.2 富途证券

1. **注册富途账号**：https://www.futunn.com
2. **开通模拟盘**：富途 → 模拟交易
3. **获取 OpenD API Token**
4. **配置到系统**

## 8. 故障排查

### 8.1 服务无法启动

```bash
# 1. 检查 Docker 是否运行
docker info

# 2. 查看详细日志
docker-compose logs -f

# 3. 清理重新启动
docker-compose down
docker system prune -f
./start.sh
```

### 8.2 数据库连接失败

```bash
# 检查数据库容器
docker-compose ps db

# 手动连接测试
docker-compose exec db mysql -u root -p

# 查看数据库日志
docker-compose logs db
```

### 8.3 选股无结果

可能原因：
1. 非交易时间（A 股需要在交易日 15:00-17:00 运行）
2. 未配置数据源 Token
3. 当日市场无符合条件的股票

解决方案：
- 配置 AKShare Token（免费）
- 在交易时间内运行
- 使用历史数据回测验证策略

### 8.4 回测报错

```bash
# 查看回测日志
docker-compose logs backend | grep backtest

# 检查数据是否完整
docker-compose exec backend python -c "from backend.app.crud import xxx; ..."
```

### 8.5 API 调用超时

```bash
# 检查后端资源使用
docker stats

# 增加后端资源
# 编辑 docker-compose.yml:
# backend:
#   deploy:
#     resources:
#       limits:
```
