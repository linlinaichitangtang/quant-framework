# "OpenClaw决策 + FMZ执行" 量化交易策略文档

## 策略总览

### 架构理念
- **决策层：** OpenClaw作为策略大脑，负责市场数据分析、选股、信号生成
- **执行层：** FMZ（发明者量化）作为交易手脚，负责订单执行、仓位管理
- **数据流：** 市场数据 → OpenClaw分析 → 生成交易信号 → JSON API推送 → FMZ执行 → 执行结果返回OpenClaw

### 策略分类
| 市场 | 策略风格 | 昵称 | 持仓周期 | 核心逻辑 |
|------|----------|------|----------|----------|
| A股 | 超短隔夜 | 约炮式 | 1天（隔夜） | 尾盘竞价买入，博取次日冲高，不恋战 |
| 港股/美股 | 事件驱动 | 恋爱式 | 数天~数周 | 基于财报/新闻事件，持有期权等待发酵 |

## 第一部分：A股"约炮式"超短策略

### 一、选股规则

#### 1. 技术形态筛选
- **股价位置：** 股价站在20日均线之上（中期趋势向上）
- **当日涨幅：** 3% < 当日涨幅 < 8%（排除涨停无法介入，排除涨幅不足无强度）
- **分时形态：** 尾盘半小时拉升，均线多头排列，收盘价接近当日最高点
- **振幅要求：** 当日振幅 > 4%（保证足够的活跃度）

#### 2. 量能要求
- **当日成交量：** 大于5日均量的1.5倍（放量上涨，资金介入明显）
- **换手率：** 3% < 换手率 < 20%（排除过低不活跃，过高出货嫌疑）
- **量比：** 量比 > 1.2（相对放大，今日强于昨日）

#### 3. 趋势要求
- **短期趋势：** 近3天累计上涨（短期强势）
- **中期趋势：** 20日均线向上（趋势向上，减少下跌风险）
- **无重大利空：** 近3天无利空公告（排除黑天鹅）

#### 4. 时间窗口
- **交易时间：** 仅在每个交易日14:30之后开始选股
- **避开时间：** 重大利空日、大盘指数大跌超过2%当日不操作
- **连板股：** 排除连续3个涨停以上的个股（规避高位接盘风险）

#### 5. 资金面
- **流通市值：** 50亿 < 流通市值 < 500亿（兼顾流动性和弹性）
- **行业：** 优先当日热点板块（资金聚焦方向）
- **排除：** ST股、退市风险股、当日停牌股

### 二、买入规则

#### 1. 决策时间
- **选股时间：** 交易日14:30 - 14:55
- **最终决策：** 14:55 - 14:57完成选股和信号生成
- **下单时间：** 14:57 - 14:59（尾盘集合竞价阶段下单）

#### 2. 下单方式
- **下单价格：** 以当前市价（集合竞价阶段价格）下单
- **下单数量：** 根据仓位规则计算（由风控规则确定，文档见风控仓位规则.md）
- **成交保证：** 尾盘集合竞价保证成交，不追求最低价成交

#### 3. 入选标的数量限制
- 每日最多入选3只标的
- 单只标的仓位不超过总仓位的20%（风控规则详见风控文档）

### 三、卖出规则

#### 1. 止盈条件
- **止盈目标：** 买入后次日上涨达到+2%（含手续费）
- **止盈方式：** 达到止盈目标立即市价卖出，不贪心
- **未达到止盈：** 持有到下午14:30观察，决定是否卖出

#### 2. 止损条件
- **止损目标：** 买入后次日下跌达到-2%（含手续费）
- **止损方式：** 触及止损线立即市价卖出，严格执行
- **开盘低开：** 如果开盘直接低开超过-2%，开盘立即止损卖出

#### 3. 尾盘卖出规则
- **次日14:30检查：** 既没有止盈也没有止损，14:30无论盈亏都卖出
- **不持仓过夜：** 绝对不持有超过1个交易日，严格执行隔夜策略
- **特殊情况：** 如果次日停牌，无法卖出则持有，复牌第一天卖出

#### 4. 卖出总结
| 情况 | 处理方式 |
|------|----------|
| 次日涨 ≥ +2% | 立即止盈卖出 |
| 次日跌 ≤ -2% | 立即止损卖出 |
| 开盘低开 ≥ 2% | 开盘立即止损 |
| 未触及止盈止损 | 14:30卖出 |
| 停牌 | 复牌首日卖出 |

## 第二部分：港股美股"恋爱式"事件驱动策略

### 一、策略理念
- 基于事件驱动（财报、新闻、政策），判断标的未来方向
- 使用期权杠杆，以小博大，持有到事件发酵完成
- 持仓周期数天到数周，不频繁交易，像恋爱一样相处一段再分手

### 二、事件驱动监控要点

#### 1. 事件类型筛选

| 事件类型 | 监控要点 | 触发条件 |
|----------|----------|----------|
| **财报超预期** | 季度财报营收/利润超分析师预期 | 财报公布后股价高开 > 3%（超预期上涨） |
| **财报不及预期** | 季度财报营收/利润低于分析师预期 | 财报公布后股价低开 > 3%（低于预期下跌） |
| **重大政策利好** | 行业利好政策出台，龙头标的受益 | 政策公布当日股价上涨 > 2% |
| **重大产品发布** | 科技公司发布新产品，市场反应积极 | 发布后股价上涨 > 2% |
| **黑天鹅事件** | 公司负面新闻、违规调查、创始人问题 | 事件公布后股价下跌 > 5% |

#### 2. 监控频率
- **财报季（每个季度初）：** 每日监控盘前财报，盘中实时监控
- **非财报季：** 每日盘前盘后各监控一次新闻
- **舆情监控：** 监控主流财经媒体、Twitter、公告信息

#### 3. 标的筛选规则
- **流动性：** 日均成交量 > 1000万美元（美股）/ > 1000万港币（港股）
- **期权流动性：** 有活跃期权交易，买卖价差 < 0.5美元
- **市值：** 市值 > 20亿美元（避免仙股）
- **趋势：** 事件发生后趋势明确，沿着事件方向运行

### 三、期权策略应用场景

根据不同事件和方向，选择不同期权策略：

#### 1. Long Call（买入看涨期权）
**应用场景：**
- 财报超预期，明确看多
- 重大政策利好，行业龙头受益
- 股价突破关键阻力位，事件确认上涨趋势

**选择合约规则：**
- 行权价：选择略高于当前股价的实值或平值合约
- 到期日：选择4周左右到期（给事件足够发酵时间）
- 仓位：单策略不超过总仓位10%（期权高杠杆）

**入场出场：**
- 入场：事件确认后，当日入场
- 止盈：标的价格上涨达到预期，期权翻倍或涨50%以上可部分止盈
- 止损：期权亏损达到50%，止损离场
- 到期：无论盈亏，到期前一周卖出平仓，不行权

#### 2. Long Put（买入看跌期权）
**应用场景：**
- 财报不及预期，明确看空
- 重大负面事件，公司基本面受损
- 股价跌破关键支撑位，确认下跌趋势

**选择合约规则：**
- 行权价：选择略低于当前股价的实值或平值合约
- 到期日：选择4周左右到期
- 仓位：单策略不超过总仓位10%

**入场出场：**
- 入场：事件确认后，当日入场
- 止盈：标的价格下跌达到预期，期权翻倍或涨50%以上部分止盈
- 止损：期权亏损达到50%，止损离场
- 到期：到期前一周卖出平仓

#### 3. 跨式策略（Straddle）：同时买Call和Put
**应用场景：**
- 财报即将公布，预计波动率将大幅放大，但方向不确定
- 重大事件公布前，市场等待方向选择
- 标的当前处于盘整，即将突破，无论上下都有大波动

**选择合约规则：**
- 行权价：选择当前股价的平值行权价
- 到期日：选择财报后1-2周到期（让财报波动充分体现）
- 仓位：Call和Put各占一半仓位，总仓位不超过10%

**入场出场：**
- 入场：财报发布前2-3天入场
- 止盈：波动率大幅上升后，整体盈利达到50%以上止盈
- 止损：整体亏损达到30%，止损离场
- 财报后：财报发布后，无论方向，抓住波动率上涨后获利了结

#### 4. 宽跨策略（Strangle）：买虚值Call + 虚值Put
**应用场景：**
- 预计财报后会有大波动，但认为更可能往某一侧走，想降低成本
- 标的价格即将突破大区间，方向不确定但波动确定
- 成本比跨式更低，杠杆更高

**选择合约规则：**
- 行权价：Call选择高于当前股价5%的虚值，Put选择低于当前股价5%的虚值
- 到期日：同跨式策略
- 仓位：总仓位不超过10%

#### 5. 牛市价差/熊市价差
**应用场景：**
- 对方向比较确定，但不想付出太高权利金
- 预计上涨/下跌幅度有限，控制成本
- 波动率较高时，买入期权太贵，用价差降低成本

**选择规则：**
- 牛市价差（看多）：买入低行权价Call，卖出更高行权价Call
- 熊市价差（看空）：买入高行权价Put，卖出更低行权价Put
- 到期日：4周左右
- 最大收益有限，最大亏损也有限，适合方向明确但波动率高的场景

### 四、入场出场规则总结

| 策略类型 | 入场时机 | 止盈条件 | 止损条件 | 持仓周期 |
|----------|----------|----------|----------|----------|
| Long Call | 事件确认上涨当日 | 期权盈利50%-100%部分止盈 | 期权亏损50%止损 | 1-4周 |
| Long Put | 事件确认下跌当日 | 期权盈利50%-100%部分止盈 | 期权亏损50%止损 | 1-4周 |
| 跨式 | 财报前2-3天 | 整体盈利50%+止盈 | 整体亏损30%止损 | 持有到财报后 |
| 宽跨 | 突破前入场 | 方向对了盈利100%+止盈 | 整体亏损30%止损 | 持有到突破后 |
| 价差 | 方向明确后 | 达到最大收益区域止盈 | 亏损达到权利金30%止损 | 1-4周 |

### 五、持仓原则
- **不重仓：** 单事件期权策略仓位不超过总仓位10%
- **分散：** 同时持有的事件不超过3个
- **不扛单：** 严格止损，事件逻辑被打破立即离场
- **到期前平仓：** 不持有期权到期，提前一周卖出平仓

## 第三部分：策略框架整合

### 一、信号生成格式
OpenClaw生成交易信号为JSON格式：

```json
{
  "strategy": "a股隔夜/美股事件驱动",
  "action": "buy/sell",
  "symbol": "标的代码",
  "market": "CN/HK/US",
  "price": 现价,
  "quantity": 下单数量,
  "order_type": "market/limit",
  "stop_loss": 止损价,
  "take_profit": 止盈价,
  "expire_time": 订单失效时间
}
```

### 二、执行流程
1. **A股每日：**
   - 14:30 开始选股
   - 14:57 发送买入信号到FMZ
   - 次日开盘后监控，触及止盈止损FMZ自动执行
   - 次日14:30 未触及则发送卖出信号

2. **港股美股事件：**
   - 监控到事件触发，符合条件生成信号
   - 发送期权买入信号到FMZ
   - 期间监控价格和事件逻辑
   - 触及止盈止损发送卖出信号，或到期前发送平仓信号

### 三、策略核心要点
| 项目 | A股约炮式 | 港股美股恋爱式 |
|------|-----------|----------------|
| 持仓周期 | 1天（隔夜） | 数天~数周 |
| 选股逻辑 | 技术面+量能尾盘选股 | 事件驱动+方向判断 |
| 工具 | 股票现货 | 期权 |
| 止盈止损 | 严格2% | 期权权利金50%止损 |
| 频率 | 每日可交易 | 事件驱动，有空头才做 |
| 核心 | 不恋战，快进快出 | 等事件发酵，赚趋势钱 |
