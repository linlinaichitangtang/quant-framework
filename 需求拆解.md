# "OpenClaw决策 + FMZ执行" 量化交易框架开发需求拆解

## 原始需求

皇帝陛下下达需求：
1. 架构设计：OpenClaw做决策大脑，FMZ做执行手脚，数据流：市场数据 → OpenClaw分析 → 生成交易信号 → 发信号给FMZ → FMZ执行 → 返回结果
2. A股策略：尾盘竞价买入博取次日冲高，严格2%止盈止损，"约炮式"不恋战
3. 港股/美股策略：事件驱动（财报/新闻）+ 期权策略（Long Call/Long Put/跨式等），"恋爱式"持有
4. 风控：单笔止损+仓位限制+黑天鹅防范
5. 接口：定义JSON格式API指令

## 任务分派

### 1. 麟渊（策略研究员）
**任务：** 整理完善策略逻辑，输出完整的策略文档
**具体要求：**
- A股尾盘策略：明确选股规则、买入规则、卖出规则（2%止盈止损）
- 港股/美股事件驱动策略：明确选股逻辑（财报/新闻事件筛选）、期权策略选择规则、入场/出场规则
- 整理策略文档框架，包含策略理念、适用市场、信号生成逻辑

**输出文件：** `docs/策略文档.md`

### 2. 麟察（市场监控官）
**任务：** 设计市场数据和新闻监控方案
**具体要求：**
- A股市场：需要哪些行情数据（日线/分时/盘口），数据来源
- 港股/美股：行情数据获取方案，新闻/财报事件监控方案（数据源、监控频率、触发条件）
- 数据推送流程：如何将数据传递给OpenClaw决策大脑

**输出文件：** `docs/市场监控方案.md`

### 3. 麟澈（风控分析师）
**任务：** 制定详细的风控和仓位规则
**具体要求：**
- 单笔交易止损规则（明确针对A股/港股/美股的不同止损方式）
- 仓位管理规则：单票仓位限制、总仓位控制、不同市场仓位分配
- 黑天鹅防范机制：极端行情应对、单日跌幅限制、整体回撤控制
- 结合需求中的"单笔止损+仓位限制+黑天鹅防范"进行细化

**输出文件：** `docs/风控仓位规则.md`

## 汇总要求

各位完成后，麟御汇总完整分析结果，交还给麟昭（总参谋长），由总参谋长分派给麟工部门进行代码实现。
